# Customizing

This chapter covers low-code configuration: schema, forms, processes, and the conventions that
keep a customization consistent across customizers, projects, and time.

It assumes the [publisher and prefix](../architecture/publisher-and-prefix.md) for the project
has already been decided — everything below uses `prfx_` as a placeholder for it.

- [Naming Conventions](naming-conventions.md) — the schema naming patterns for every component
  type.
- [Tables, Columns & Relationships](tables-columns-relationships.md)
- [Forms, Views & BPF](forms-views-bpf.md)
- [Command Bar](command-bar.md)
- [Themes & Icons](themes-and-icons.md)
- [Env Variables & Connection References](envvars-connectionrefs.md)
- [Security Model](security-model.md)

!!! note "Customer standards take precedence"
    As stated in [Scope & Principles](../foundation/scope-and-principles.md): if the customer
    already has a defined naming or customizing standard, follow that instead, and document the
    deviation from this guideline.
